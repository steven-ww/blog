Java EE on Azure

Java Azure

I’ve just spent some time with Microsoft Azure and their App Service.

This gives you the ability to deploy to a provisioned environment, in this case, Tomcat or Wildfly (but others like Java SE, Nodejs and PHP are supported).

What this service allows you to do, is with nothing more than a maven POM file and an Azure plugin, deploy your WAR or EAR to an environment that’s spun up for you as you deploy.

No need to create a virtual server and install WildFly.

Also, the certificate and domain are handled for you. Changeable of course, but quick to get running and testing.

To configure the container, e.g. add a datasource, you can provide a script to be executed. In my case that script simply executed some WildFly CLI commands to create a Postgres datasource and point at the postgres jdbc driver.

Behind the scenes a virtual server is being created with a number of docker containers and an external volume.

The volume allows for a persistent space where your startup script and jar files are placed (Basically anything in the /home directory is on this volume.)

Containers are then provisioned which allows you to interact with this volume via ftp or ssh.

Containers are also spun up to provision your WildFly server/s against which your scripts are run.

All in all a great way to get started quickly, automate the provisioning and configuration of your environment and deployment of your code.

Definitely worth having a look at.
